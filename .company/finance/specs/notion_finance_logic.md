# 📑 Notion 財務システム：機能・ロジック仕様書 (Legacy Spec)

本ドキュメントは、Notion上に構築されていた既存の財務システムの機能を詳細に明文化したものです。新システム（AI-Company Dashboard）の実装において、これら全てのロジックを忠実に再現、または進化させることを保証します。

---

## 1. 資産管理のコア・ロジック

### 1.1 決済DB（スナップショット管理）
**役割**: 月末時点での各アセットの「残高」を固定レコードとして保存する。

| 機能名 | ロジック詳細 | 目的 |
| :--- | :--- | :--- |
| **合計金額算出** | `銀行口座` + `PayPay` + `へそくり` + `Suica(時計/スマホ)` + `Google/スタバ` + `財布(青/黒橙)` の総和 | 全流動資産の把握 |
| **実質貯金額 (Net Worth)** | `合計金額` - `負債合計(奨学金等)` | 借金を差し引いた真の自由資産の確認 |
| **前月増減比較** | `今月合計` - `前月合計` | 家計の収支が黒字か赤字かの判定 |
| **自動色付け** | 増減額がプラスなら「青」、マイナスなら「赤」、停滞なら「灰」 | 視覚的な財務健全性アラート |

### 1.2 リレーションシップ構成
**役割**: 異なるデータベース間の情報を統合し、多角的に分析する。

- **資産 ↔ 収支**: 支出マスターおよび収入マスターとの紐付け。
- **中間DB**: データの整合性を保つためのブリッジ（ハブ）構造。

---

## 2. 支出・収入マスターの役割

### 2.1 支出マスター (Expense Tracking)
- **カテゴリ管理**: 食費、日用品、趣味などの属性付与。
- **支払元追跡**: どの財布/口座から支出されたかの記録。

### 2.2 収入マスター (Income Tracking)
- **収入源分析**: 給与、副業、還付金などの分類。
- **未収金管理**: 入金予定日の把握。

---

## 3. 実装確認チェックリスト (新システム用)

新システム構築後、以下の項目が正しく動作しているかをCEOと共に確認します。

- [ ] **全アセット合算ロジック**: Notionと1円の狂いもなく合算されているか。
- [ ] **負債控除ロジック**: 奨学金等のマイナス項目が正しく反映されているか。
- [ ] **時系列比較**: 過去データからの増減が正しくトレースされているか。
- [ ] **UI/UX**: Notionの「決済日」を基準とした、見やすいダッシュボードになっているか。
