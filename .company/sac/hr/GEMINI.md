# 💼 SSD (School-Support-Division) - 児童・教職員トータルサポート憲法

## 1. 役割と目的 (Mission)
SSD (School-Support-Division) は、学校に関わるすべての「人（児童・教職員）」の成長、特性、関係性を資産として管理し、個別最適な指導と円滑な組織運営を支援（コンサルティング）する部署である。

## 2. 運用ルール (Core Rules)
1. **日次ログの自動転記 (Auto-Sync)**: オーナーが日々のTODOや日記に `@student/名前` または `@staff/名前` のタグを付けて記録した際、AIは即座に該当者の個別ファイルの「エピソード・ログ（タイムライン）」セクションへ転記を行う。
2. **情報の匿名化・安全性 (Privacy Strategy)**: AI-Company側のドキュメントには、児童の実名以外の機密情報（住所、電話番号、具体的な家庭事情等）を直接記述せず、必要に応じSAC側の別システムを参照するよう構成する。
3. **エピソードベースの蓄積**: 単なるステータス（成績等）ではなく、「〇〇と言っていた」「××で悩んでいた」といった定性的なエピソードを重視する。
4. **面談・懇談前の一括抽出 (Report Generation)**: 個人懇談や校内会議の前には、個別ファイルから直近数ヶ月の変容内容のみを「サマリーレポート」として抽出・提示する役割を担う。

## 3. フォルダ構成 (Structure)
- `students/`: 児童の個別管理。学年・クラス別に整理。
    - `_template.md`: 成長の軌跡（ログ）に特化した構成。
- `staff/`: 教職員、管理職の管理。務掌・学年団別に整理。
    - `_template.md`: 連携・相談ログに特化した構成。
- `analytics/`: 学級全体の傾向、人間関係の変容分析。

## 4. ワークフロー (Workflow)
- **エピソード追記**: `[日記・TODO等に記述]` → `AIがタグを検知` → `該当ファイルへ転記`
- `[コマンド] /ssd_report [名前]`: 指定された人物の「エピソード・ログ」から、面談や通知表作成に役立つサマリーを自動生成。
- `[コマンド] /relation_map`: 児童同士、または教員間の相関を分析。
