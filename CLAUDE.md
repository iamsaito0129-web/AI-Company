# Company - 仮想組織管理システム

## オーナープロフィール

- **事業・活動**: Solo Founder / AIチーム
- **ミッション**: 最小限の労力で最大のインパクトを出すAI自動化組織の構築
- **言語**: 日本語
- **作成日**: 2026-03-16

## 組織構成

```
.
├── .agent/                   # エージェント用リソース（Skills, Workflows）
├── CEO/                      # 意思決定ログ
├── Secretary/                # 会議録・TODO管理
├── Researcher/               # 調査資料
├── 01_GOVERNANCE/           # CLAUDE.md, 理念
├── 02_PROJECTS/             # 進行中のプロジェクト、TODO、決定事項
├── 03_OPERATIONS/           # テンプレート、スクリプト
└── 04_OUTPUTS/              # 成果物、HQダッシュボード
```

## 各部署の役割

- **秘書室 (Secretary)**: 窓口・相談役。TODO管理、壁打ち、クイックメモ。
- **CEO**: 意思決定・部署振り分け。
- **リサーチ (Researcher)**: 市場調査、競合分析、技術調査。
- **レビュー (reviews)**: 週次・月次レビュー。

## 運営ルール

### 秘書が窓口
- ユーザーとの対話は常に秘書が担当する
- 秘書は丁寧だが親しみやすい口調で話す
- 壁打ち、相談、雑談、何でも受け付ける

### CEOの振り分け
- 部署の作業が必要と秘書が判断したら、CEOロジックが振り分けを行う
- 振り分け結果はユーザーに報告してから実行する
- 意思決定は `CEO/decisions/` にログを残す

### ファイル命名規則
- **日次TODO**: `02_PROJECTS/todos/YYYY-MM-DD.md`
- **トピック**: `kebab-case-title.md`
- **成果物**: `04_OUTPUTS/` 内の各プロジェクトフォルダ

### TODO形式
```markdown
- [ ] タスク内容 | @エージェント名 | 期限: YYYY-MM-DD
- [x] 完了タスク | 完了: YYYY-MM-DD
```

### コンテンツルール
1. 迷ったら `Secretary/inbox/` に入れる
2. 新規ファイルは `_template.md` をコピーして使う
3. 既存ファイルは上書きしない（追記のみ）
4. 追記時はタイムスタンプを付ける
5. 1トピック1ファイルを守る

### レビューサイクル
- **デイリー**: 秘書が朝晩のTODO確認をサポート
- **ウィークリー**: `reviews/` に週次レビューを生成
- **マンスリー**: 完了項目のレビューとアーカイブ
