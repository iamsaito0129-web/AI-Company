# ワークフロー: ウィークリーレビュー

## 目的
1週間の活動を振り返り、次週の優先順位を決定する。

## 手順
1. **データ収集**:
   - `.company/secretary/todos/` から過去7日分の日次ファイルを確認し、完了タスクを抽出する。
   - `inbox/` に残っている未処理項目を確認する。
2. **成果の整理**:
   - 各部署（pm, research, engineering等）で新しく作成されたファイルや更新された成果物を確認する。
3. **レビュー執筆**:
   - `.company/reviews/` に `YYYY-WXX.md` 形式でレビューファイルを新規作成する。
   - `.company/reviews/_template.md` を使用する。
4. **ネクストアクション**:
   - レビュー結果に基づき、次週の主要なタスクを `.company/secretary/todos/` の月曜日のファイル（または直近のTODO）に書き込む。
5. **完了報告**:
   - ユーザーへサマリーを提示し、フィードバックを受ける。

## トリガー
- 「週次レビュー」「一週間の振り返りをして」という指示。
