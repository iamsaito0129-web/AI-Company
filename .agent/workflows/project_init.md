# ワークフロー: プロジェクト・イニシエーション

## 手順
1. **ヒアリング**:
   - プロジェクトの名前、目的、納期、関わる部署を確認する。
2. **フォルダ・ファイル作成**:
   - `pm/projects/` にプロジェクト管理ファイル（`[project-name].md`）を作成。
   - `pm/tickets/` に最初のタスクチケットを数件作成。
   - 必要に応じて `research/topics/` や `marketing/content-plan/` に該当プロジェクト用のファイルを作成。
3. **CEOへの報告**:
   - `ceo/decisions/` にプロジェクト開始の決定事項を記録する。
4. **マイルストーン設定**:
   - `GEMINI.md` の現在のプロジェクト状況を更新し、全体のタイムラインを可視化する。
5. **オーナーへの完了報告**:
   - 準備が整ったファイルへのリンクを一覧で提示する。

## トリガー
- 「新規プロジェクト開始」「[名前] というプロジェクトを立ち上げて」という指示。
