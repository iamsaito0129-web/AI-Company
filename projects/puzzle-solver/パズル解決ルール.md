# 3D Route Puzzle 解決ルール

このドキュメントでは、5x5x10の直方体パズルを解決するための基本ルールと仕様をまとめます。

## 1. 基本設定
- **形状**: 5x5x10 の直方体（表面積 250マス）。
- **ID割り当て**: 立体表面の全250面に対し、1から250の一意なIDが割り振られています。
- **データ管理**: すべてのマスの状態（スタート、ゴール、壁、曲がり角）は `puzzle_config.json` で管理されます。

## 2. マスの種類と意味
| 色 | 役割 | ルール |
| :--- | :--- | :--- |
| 🟢 **緑 (START)** | スタート地点 | ルートの開始位置です。 |
| 🔴 **赤 (GOAL)** | ゴール地点 | すべての「黄色」を経て最後に到達する地点です。 |
| 🟡 **黄 (YELLOW)** | 必須通過地点 | **必須通過**です。曲がる必要はありませんが、必ずルートに含める必要があります。 |
| 🔵 **水色 (LIGHTBLUE)** | 障害物 / 壁 | **通行禁止**です。ルートに含めることはできません。 |
| 🟣 **紫 (ROUTE)** | 移動経路 | 最適化された解答ルートです。矢印で方向が表示されます。 |

## 3. 移動と探索のルール
- **曲がる回数の制限**: 
    - ルート全体で**合計16回**曲がらなければなりません（少なすぎても多すぎてもいけません）。
    - 直進（同じ方向への移動）はカウントしません。
- **ワープとカウント**:
    - 展開図上の端を越えて物理的に離れた場所に移動する「ワープ」自体は、曲がるとカウントしません。
    - ただし、**ワープした直後に進行方向を変える場合**は、通常の曲がると同様に1回とカウントします。
- **黄色マスの扱い**:
    - 黄色マスは必ず通過しなければいけませんが、そこで曲がる必要はありません。直進して通過することも可能です。
- **面を跨ぐ移動**: 直方体の角や辺に達した際、隣接する別の面にスムーズにワープして移動します。
- **通過制限**: 同じマスを二度通ることはできません（自己交叉禁止）。

## 4. UI（展開図ビューア）の仕様
- **正しい展開図**: 十字型の構成で、上面・下面・側面4面がそのトポロジー（接続関係）を保って配置されます。
- **ワープ表示**: 展開図上で物理的に離れているが3D的に繋がっているマスへの移動は、`🌀` アイコンで表示されます。
- **動作モード**:
    - **自動反映モード**: ソルバーが算出した解を自動的に描画します。
    - **ユーザー操作モード**: ユーザーが自分で経路を指定・修正できます（矢印をクリックで向きを変更）。

## 5. 運用ワークフロー
1. `puzzle_config.json` で初期マス配置を定義。
2. `puzzle_solver_solve.py` を実行して解を探索。
3. `3D_Route_Puzzle_Unfolded_v3.html` をブラウザで開き、結果を確認・分析。
