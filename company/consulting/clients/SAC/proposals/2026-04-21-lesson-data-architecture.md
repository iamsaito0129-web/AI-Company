# 🎖️ SAC コンサルティング報告書：授業データ管理・ナビゲーション構成の最適化

## 1. 現状の課題 (As-Is)
- 挨拶や表紙など、毎回共通して使用するスライド資料を重複して作成している。
- 固定資料と「授業の中身（流動的な部分）」が混在しており、メンテナンス性が低い。
- スライド間の移動が手動であり、戻る操作や特定のページへのジャンプが煩雑。

## 2. 理想の状態 (To-Be)
- **基盤と中身の分離**: 固定資料（テンプレート）と、各授業固有のコンテンツを完全に分離する。
- **ノンリニア・ナビゲーション**: ObsidianのCanvas機能や、リンクベースのボタンを利用し、ワンクリックで「表紙」「挨拶」「本編」を行き来できる構造。
- **データ駆動型更新**: 授業の中身だけを入れ替えれば、共通部分は自動的に再利用される仕組み。

## 3. 具体的なアクションプラン (Action)

### 3-1. ディレクトリ構造の再定義（提案）
SAC環境の `lessons/` ディレクトリ配下を以下のように構成することを推奨します。

```text
lessons/
├── _common/                 # 共通資料（固定）
│   ├── 01_greeting.md       # 最初と最後の挨拶
│   ├── 02_cover_template.md # 表紙テンプレート
│   └── 03_rules.md          # 授業の約束事
├── subjects/                # 教科学習（流動的）
│   └── english/
│       └── G5/
│           └── U01/
│               ├── contents/     # 授業の本編（スライドの中身）
│               └── dashboard.md  # このUnitの司令塔（Canvasまたはリンク集）
└── navigator.canvas         # 視覚的な授業の流れ管理ツール
```

### 3-2. スライド・ナビゲーションの実装案
Obsidianの **「Callouts」** または **「Button plugin」**（導入済みの場合）を活用し、共通資料へのポータルを作成します。

**実装例 (dashboard.md):**
```markdown
> [!IMPORTANT] 授業コントロールパネル
> [[01_greeting|🎬 挨拶を開始]] | [[lessons/subjects/english/G5/U01/contents/L01|📖 本編(L1)に進む]] | [[03_rules|⚖️ ルールの確認]]
```

### 3-3. リンクによる「戻る」ボタンの設置
共通スライドの末尾に `[◀ 前の画面に戻る](javascript:history.back())` または `[[dashboard|🏠 ダッシュボードへ戻る]]` を配置することで、操作ミスを防ぎます。

## 4. 今後のロードマップ

2. **フェーズ2**: 各教科ディレクトリへの「戻るボタン」付きテンプレートの配布。
3. **フェーズ3**: 自動化スクリプトによる、日付と教科に基づいた授業ダッシュボードの自動生成（将来）。
