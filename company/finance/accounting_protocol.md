# 💰 AI-Company 財務部：2026年度 次世代デジタル財務プロトコル (v2.0)

## 1. 目的と新方針
AI-Company CEO（彩人様）の資産管理をNotionから**「AI-Company内自律型システム」**に完全に統合する。
Notionを「旧システム」として廃止し、今後はリポジトリ内のデータ（GitHub、ローカルMarkdown）とカスタマイズされたエンジニアリング・ダッシュボードで運用を完遂する。

- **コアコンセプト**: **「Notionからの完全独立と、AIによる高度な意思決定の自動化」**。
- **引き継ぎ要件**: 2025年度以前の全履歴を引き継ぎ、カンパニー内の資産として集約する。

## 2. 刷新されたシステム構造（次世代管理）
- **データ拠点**: `.company/finance/data/`
    - `master/`: Notionから抽出・変換された過去のマスターデータ（JSON/CSV形式）。
    - `records/`: 3月分以降の新規入力レコード（Markdown形式）。
- **集計エンジン**: `.company/finance/scripts/`
    - Python/JSによる自動集計スクリプト。
- **意思決定ビュー（UI）**: 
    - カスタムHTML/JSダッシュボード `Finance-Dash` による視覚化（Wow Factorの実装）。

## 3. 次世代決算ワークフロー
Notionの手入力から解放された、AI-Company独自のワークフロー。

1. **データ収集 (Data Acquisition)**:
    - PDF（給与明細）、CSV（銀行・カード明細）を `.company/finance/invoices/` に配置。
    - **楽天証券データ**: 毎月、AI（私）が証券評価額をヒアリングし、自動反映（3月分より開始）。
2. **AI処理 (AI Processing)**:
    - スクリプトが全データをスキャンし、カテゴリの自動判別と `data/records/` へのMarkdown出力を行う。
3. **レビュー・承認 (Review & Approval)**:
    - 生成された月次レポートをCEOが承認し、月次決算を確定。

## 4. 証券・投資管理プロトコル（新設）
- **楽天証券**:
    - 評価額総計と、前月比（損益率）を定点観測。
- **戦略**:
    - 「質素倹約」により産出された余剰資金の、特定銘柄への自動アロケーション（投資アドバイス機能の追加予定）。
