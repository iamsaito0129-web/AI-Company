# 📊 資産管理アプリ (Finance Dash) 開発状況レポート

現在開発中のプレミアム財務管理システム（AI-FINANCE / Finance Dash v2）について、最新の意思決定を反映した開発状況と今後の詳細な手順をまとめました。

## 1. 🟢 開発方針の重要なアップデート（ピボット）

直近の協議により、以下の重要な方針転換が決定しました。

1.  **Notionからの完全独立 (No Notion API)**
    *   Notion APIとの連携は行わず、**Notionからの完全脱却**を実施します。
    *   既存データはNotionからエクスポートしたCSVを活用し、当システム（ローカルJSON等）を唯一のマスターデータとして独立稼働させます。
2.  **モバイルファースト至上主義 (Mobile-First as Top Priority)**
    *   開発のいかなる状況においても、**スマホからの操作性・視認性を最優先**とします。
    *   外出先で1秒で起動し、片手で瞬時に支出入力や残高確認ができるUI/UXを徹底します。
3.  **優先順位の変更**
    *   【最優先】 スマホ最適化、動的 Safe-to-Spend 計算、給与監査モジュールの実装。
    *   【優先度低】 ACT (AI-Company Token) システムの実装は後回しとします。

---

## 2. 🟢 現在の達成状況・実装済み項目

*   **UI/UXの基礎構築 (Premium Aesthetics)**
    *   Glassmorphism（グラスモーフィズム）を採用した透明感のあるパネルデザイン。
    *   ドットマトリクスフォント、アクセントカラー「赤いENTボタン」等のインダストリアルデザイン要素。
*   **バックエンド・データ基盤**
    *   Node.jsローカルサーバー (`server.js`) の稼働。
    *   JSONマスターデータ (`finance_master.json`) のスキーマ定義。

---

## 3. 🟡 優先実施タスクと詳細な手順・提案

方針転換に基づき、以下の手順で開発を進めます。

### 🚀 優先度1: モバイル最適化の徹底（Vite + PWA導入）
スマホでの「1秒起動」と「ネイティブアプリのような操作感」を実現するため、現在のHTMLベタ書きからモダンなビルド環境へ移行します。

*   **具体的な手順**:
    1.  `npm create vite@latest` でプロジェクトの雛形を再構築（ReactまたはVanilla TypeScript）。
    2.  既存のデザイン（CSS/Glassmorphism）を移植。
    3.  `vite-plugin-pwa` を導入し、ホーム画面に追加可能（オフライン対応・キャッシュ有効化）な **PWA (Progressive Web App)** に設定。
*   **提案（Premium Aesthetics）**:
    *   モバイル画面下部に固定される「Bottom Navigation Bar（グラスモーフィズム）」を配置し、親指だけで「ダッシュボード」「入力」「給与監査」を切り替えられるようにします。

### 🚀 優先度2: 動的 Safe-to-Spend 計算の完全実装
Notionなしで、日々の予算管理を自動化します。

*   **具体的な手順**:
    1.  NotionからエクスポートしたCSVを `finance_master.json` にパース・結合する初期インポートスクリプトを実行。
    2.  `server.js` (またはフロントのロジック) にて、「今月の固定予算」－「今月の累計支出」＝「残額」を計算。
    3.  「残額」÷「今月の残り日数」＝ **「今日使える額 (Safe-to-Spend)」** をリアルタイム算出。
*   **提案（UX）**:
    *   スマホのテンキー（赤いENTボタン）で支出を入力した瞬間、ローカルのJSONが更新され、Safe-to-Spendの数字がパラパラッと減るアニメーション（オドメーター効果）を実装します。

### 🚀 優先度3: 給与監査 (Salary Audit) モジュールの実装
教員特有の複雑な給与体系を毎月検証・記録する専用UIを構築します。

*   **具体的な手順**:
    1.  モバイル画面で縦スクロールしやすい「アコーディオン型」の入力フォームを作成。
    2.  入力項目：額面（基本給、地域手当、義務教育等教員特別手当など）、控除項目（税金、共済年金など）。
    3.  過去のデータや基本給のテーブルと比較し、**「予期せぬ天引きの変動」や「手当の漏れ」がないかを自動計算してアラート**を出すロジックを実装。
*   **提案**:
    *   毎月入力する手間を省くため、「先月と同じコピーして差分だけ修正する」ボタン（Duplicate from Last Month）を設置します。
