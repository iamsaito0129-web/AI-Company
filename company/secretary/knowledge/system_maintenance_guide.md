# 🛠️ システムメンテナンス・ガイドライン

## 1. トークン・認証管理 (Token Management)

| サービス | 更新頻度 | 管理ツール / コマンド | 備考 |
| :--- | :--- | :--- | :--- |
| **Google Calendar** | **7日** (Testing時) / 永続 (Production時) | `npm run auth` | 現在テストモードの場合、1週間でリフレッシュトークンが失効します。 |
| **GitHub PAT** | 設定による (通常30-90日) | GitHub Settings > Developer settings | `GITHUB_TOKEN` の有効期限。失効前に通知が届きます。 |
| **Notion API Key** | 永続 (再生成まで) | Notion Integrations My integrations | 基本的に更新不要ですが、権限変更時に再生成が必要な場合があります。 |
| **MCP Server Config** | 随時 | `.gemini/antigravity/mcp_config.json` | サーバーパスの変更や引数の追加時に編集。 |

### ⚠️ Google Calendar トークンの失効について
昨日再認証したにもかかわらず本日失効していた原因として、以下の可能性が考えられます：
1. **GCPプロジェクトの状態**: プロジェクトが「Testing」モードのままで、かつ外部ユーザータイプの場合、リフレッシュトークンの寿命は7日間に制限されます。
2. **スコープの変更**: MCP設定ファイル側で要求しているスコープが、前回の認証時と異なっている場合。
3. **ローカルストレージの不具合**: トークンを保存する `.tokens` ファイル等が、クリーンアッププロセスなどで削除・移動された可能性。

**推奨アクション**:
GCPコンソールでプロジェクトを「Production」に移行することを検討してください（審査は不要な簡易的なものです）。

## 2. 定期的なデータメンテナンス (Data Maintenance)

| 頻度 | 項目 | 手順 |
| :--- | :--- | :--- |
| **毎日** | 翌日のTODO生成 | 20:00〜22:00頃に `Secretary` が翌日の雛形を作成。 |
| **毎週** | 週次レビュー (reviews/) | 日曜夜または月曜朝に実施。未完了タスクの棚卸し。 |
| **毎月** | 受信箱 (inbox/) の整理 | 溜まったクイックキャプチャを各部署へ完全移行。 |
| **随時** | `knowledge_base.md` 更新 | 新しく発見したオーナーの「癖」や「好み」を反映。 |

## 3. GitHub / LINE Bridge メンテナンス

- **監視項目**: GitHub Actions の実行ログ、Issueへのコメント通知。
- **対応依頼**: エラー発生時は `Engineering` 部署のデバッグログを参照。
