# AI-Company 資産管理システム仕様書

## 1. 概要
本文書は、ユーザーがNotion上に構築した高度な個人・事業資産管理システムの構造、ロジック、および運用方法を体系化したものである。
本システムの更新および監査は、AI-Company 財務部/管理部署が担当し、正確性と即時性を担保する。

## 2. データベース構造
資産管理システムは以下の4つのデータベースと1つのハブデータベースで構成される。

| データベース名 | 役割 | ID |
| :--- | :--- | :--- |
| **資産管理_収入マスター** | 収入項目、自動給与計算、支払日管理 | `1b1c6c25-58a0-80ee-b0a8-cec125b9b417` |
| **資産管理_支出マスター** | カテゴリー別支出、レシート・備考管理 | `1b1c6c25-58a0-8028-8400-f6575c0d363c` |
| **合計管理** | 期間・カテゴリー別集計レポート生成 | `1b2c6c25-58a0-80e8-81a9-f2f796c82405` |
| **中間DateBase** | 各DBを紐付けるリレーションハブ | `1b1c6c25-58a0-8081-9aa8-d601016f5fd0` |

## 3. 主要ロジック分析
### 3-1. 収入計算ロジック（給与自動計算）
`収入マスター`には、勤務先（馬渕個別等）の給与単価改定履歴がフォーミュラ内に直書きされている。
- **特徴**: 支払日（`ISln`）に基づき、IFS関数でレベル1〜5の単価を自動判別し、授業時間/事務時間と掛け合わせて算出。
- **重要点**: 2022年4月〜2025年11月までの歴史的な単価テーブルを保持。

### 3-2. レポーティング・エンジン
`合計管理`DBは、JavaScriptライクな高度なフォーミュラを搭載し、以下の機能を提供する。
- **柔軟なフィルタリング**: 指定開始日〜終了日の範囲内にあるデータのみを合算。
- **動的統計**: 選択した「計算方法」（合計、平均、最大、最小、個数）に応じて、数値を動的に表示。
- **フォーマット出力**: 数値の3桁区切り（カンマ）処理までフォーミュラで実現。

## 4. 運用・管理ルール (引き継ぎ事項)
管理部署は、以下のタスクを定期的に遂行する。
1. **マスタ情報の保守**: 給与体系の変更時、Notionフォーミュラの単価テーブルを更新する。
2. **収支の整合性監査**: 各支出項目の「カテゴリー」設定が適切か、重複がないかを確認する。
3. **レポーティング**: 定期的に「合計管理」DBの数値を確認し、異常値（高支出等）があればCEOへ報告する。
4. **マインドセットの維持**: 関連ページ「質素倹約」の内容を尊重し、不要不急の支出を抑制する提案を行う。
